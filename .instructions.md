# George's Blog & Thoughts - Custom Instructions

## 🎯 Site Overview & Purpose

**Website Title**: George — Every Word Has Weight

This is a personal blog and thought platform for publishing short-form, deeply meaningful content. Articles are:
- **1-2 minute reads** (approximately 250-500 words)
- **Simple yet heart-gripping** - emotionally resonant and impactful
- **Focused on lessons, thoughts, and reflections** - not tutorials or how-tos
- **Designed for busy readers** who want substance without fluff

---

## ✍️ Content & Writing Guidelines

### Article Characteristics
- **Brevity**: Keep to 250-500 words per article
- **Emotional Resonance**: Touch the heart while engaging the mind
- **Clarity**: Write in simple, accessible language (Grade 8-10 level)
- **Authenticity**: Share genuine thoughts, lessons, and personal insights
- **Originality**: No plagiarism; all content must be original work
- **Structure**: Hook → Insight → Reflection → Call-to-action or takeaway

### Content Types
- Personal reflections and life lessons
- Spiritual or philosophical thoughts
- Short observations about human nature
- Bite-sized wisdom and perspectives
- Stories with meaningful conclusions

### Metadata Requirements
- Compelling, specific title (not generic)
- Author byline with publication date
- Optional topic/category tag
- 1-2 sentence excerpt for preview

## 🔧 Development & Git Workflow

### Commit Requirements
**EVERY change must be committed to GitHub with clear commit messages.**

#### Commit Message Format
```
[TYPE] Brief description

Optional detailed explanation of changes and why.

Related to: [issue number if applicable]
```

#### Commit Types
- `[FEATURE]` - New article, widget, or feature
- `[FIX]` - Bug fix or correction
- `[STYLE]` - CSS/design updates
- `[CONTENT]` - Article text or metadata updates
- `[TEST]` - Test-related changes
- `[DOCS]` - Documentation updates
- `[REFACTOR]` - Code restructuring without feature changes

#### Example Commits
```
[FEATURE] Add new article: "Finding Light in Silence"

- Published new 1-minute read about stillness
- Added tags: reflection, peace
- Updated article grid and featured section

[FIX] Correct typography spacing in article body
[STYLE] Adjust dark mode colors for better contrast
[TEST] Add unit tests for theme toggle functionality
```

### Branching Strategy (Recommended)
- `main` - Production ready
- `staging` - Testing and review
- `feature/article-[title]` - New articles
- `feature/[feature-name]` - New features
- `fix/[issue-name]` - Bug fixes

## ✅ Testing Requirements

### Test Coverage
All code changes must include tests. Aim for:
- **Unit Tests**: 80%+ coverage for business logic
- **Integration Tests**: Core workflows (theme toggle, navigation, etc.)
- **Visual Regression Tests**: Ensure layout stays intact

### Test Types to Implement

#### 1. **Content Tests**
- Article metadata validation (title, date, author present)
- Article word count verification (250-500 words)
- SEO metadata checks (og:title, description, etc.)
- Link validity checks

#### 2. **Functionality Tests**
- Theme toggle switches correctly
- Navigation links work
- Mobile menu responsive
- Article grid displays properly
- Sidebar widgets render

#### 3. **Performance Tests**
- Page load time acceptable (<3s)
- Images optimized
- CSS/JS minified

#### 4. **Visual Tests**
- Layout maintained on mobile, tablet, desktop
- Colors match design spec
- Typography renders correctly
- No layout shifts

### Testing Tools (Recommended)
```json
{
  "testing": {
    "unit": "Jest or Vitest",
    "integration": "Cypress or Playwright",
    "visualRegression": "Percy or Chromatic",
    "performance": "Lighthouse CI",
    "accessibility": "Pa11y or Axe"
  }
}
```

### Running Tests
```bash
npm test                    # All tests
npm run test:unit          # Unit tests only
npm run test:integration   # Integration tests
npm run test:visual        # Visual regression tests
```

## 📝 Article Publication Workflow

### Steps to Add a New Article
1. Create article content (write in Markdown or plain text)
2. Add to HTML structure with proper metadata
3. Update article grid or featured section
4. Add tests for new content
5. Test on mobile and desktop
6. Commit with `[FEATURE] Add new article: "[Title]"`
7. Push to staging for review
8. Merge to main when approved
9. Deploy

### Article Metadata Template
```html
<div class="a-card">
  <h3 class="a-title">[Article Title]</h3>
  <p class="a-excerpt">[1-2 sentence excerpt]</p>

    <span class="a-read-time">[X min read]</span>

</div>
```

## 💡 Best Practices for This Type of Website

### Content Best Practices
1. **Quality Over Quantity** - One powerful article beats 10 mediocre ones
2. **Authenticity** - Readers feel sincerity; never fake emotion or insight
3. **Story-Driven** - Use anecdotes and examples, not abstract concepts
4. **Clear Takeaway** - Readers should understand the lesson/message
5. **Proofreading** - Perfect grammar and spelling (every word matters)
6. **Consistency** - Publish on a regular schedule (1-2x per week)

### Design Best Practices
1. **White Space** - Let content breathe; don't clutter
2. **Typography** - Excellent typography enhances emotional impact
3. **Theme Toggle** - Respect dark mode preferences
4. **Performance** - Fast loading respects reader's time
5. **Accessibility** - Color contrast, readable fonts, keyboard navigation
6. **Mobile-First** - Most readers come from mobile

### User Experience Best Practices
1. **Easy Navigation** - Find articles quickly
2. **Clear Hierarchy** - Featured article stands out
3. **Related Content** - Suggest similar articles
4. **Social Sharing** - Make articles easy to share
5. **Comments Section** - Optional, but builds community
6. **Email Newsletter** - Notify subscribers of new articles

### SEO Best Practices
1. **Descriptive Titles** - Clear, keyword-rich article titles
2. **Meta Descriptions** - Compelling 160-character descriptions
3. **Open Graph** - Proper og:title, og:description, og:image tags
4. **URL Slugs** - SEO-friendly URLs (e.g., `/articles/finding-light-in-silence`)
5. **Internal Linking** - Link to related articles
6. **Mobile-Optimized** - Mobile-first indexing is Google standard
